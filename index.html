JS变量提升

什么是变量提升

  • 是一个阶段 在代码执行之前做的事情
  • 范围：在当前作用域下

  • 含义：在当前作用域下，代码执行之前，对带var关键字和function关键字的进行声明和定义
    对var关键字是声明 var num;
    对function关键字是声明+定义 var num = 10;

    函数定义的步骤：
    1.首先开辟一个堆内存，假设引用地址FFF000;
    2.函数体的内容以字符串的形式保存在这个内存空间里
    3.将引用地址FFF000赋值给函数名fn,fn就代表了整个函数

    函数执行步骤：
    1.形成私有作用域
    2.形参赋值
    3.变量提升
    4.代码从上往下运行

三、前后端的交互流程

在浏览器的地址栏中输入url地址并按回车，这个过程发生了什么？
1、通过http协议，客户端向服务器端发起一个请求
2、查看url地址中的域名部分，将域名解析到服务器IP地址，找到对应的服务器
3、查看端口号，找到对应的服务，在这个服务中通过路径找到对应的页面，
4、服务器将对应的内容通过response响应，返回给客户端，
5、客户端拿到数据后，按以下步骤将数据在页面显示出来：
① 将所有标签生成DOM Tree
② 代码自上而下执行，当遇到script、link、img…..这些标签，会按照上面步骤重新向服务器请求
③ js部分有动态绑定的数据，进行动态绑定数据处理（ejs模板 / 字符串拼接的方式）
④ 浏览器进行渲染和绘制

什么是http事务
客户端请求+服务器响应成功，这样完整过程称为一条http事务
客户端向服务器发起请求时会携带一些数据，服务器端响应时也会向客户端发送一些数据，这些数据详细的信息称为http报文

四、详细讲解url地址

URL：统一资源定位符
URI：统一资源标识符=URL+URN，通常指的就是url
URN：统一资源名称（用的很少）

1、协议：
① http 超文本传输协议 默认端口号：80
② https http+ssl多了一个加密协议（通常网站涉及到支付或者用户隐私等） 默认端口号：8080
③ ftp 文件传输协议 默认端口号：21

2、域名：
① 一级域名 www.zhufeng.cn
② 二级域名 online.www.zhufeng.cn
③ 三级域名 html5train.online.www.zhufeng.cn

3、端口号：
相当于房间号 范围是0~65535

4、 路径：（端口号后面，问号前面）
通过路径从服务器查找对应的文件
course/index.html

5、问号传参
格式：key=value 每个参数之间一&连接
?name=zf&age=9
① 把客户端数据通过传参方式发送给服务器端
② 让服务器端返回指定参数的数据

6、hash值（跟http事务无关，跟锚点定位或路由跳转有关）

七、ajax（async javascript and xml）

在不刷新页面的前提下，局部更新数据

（1）请求的方式
get 向服务器获取响应（主体+头）的内容
head 向服务器获取响应头的内容
post 把发送内容，放在请求主体里发送给服务器，通过增加内容时用post方式
put 把发送内容，放在请求主体里发送给服务器，返回的状态码只有202，通常修改内容时，用put方式
options 是一种探测性请求，客户端和服务器端连接后，查看服务器端支持的请求方式有哪些，然后再发起支持请求方式的请求
delete 向服务器发起请求，让服务器删除某个内容

【Get系列】get | head | delete
【post系列】post | put
【get系列请求和post系列请求有何不同？】
① get系列数据是放在url地址后面发送给服务器，post系列是把数据放在请求体里面发送给服务器
② 发送的数据大小有区别，get系列 Chrome 8kb ，firefox 7kb，ie 2kb；post系列没有大小限制
③ get系列会有缓存问题，post系列没有
④ post系列请求方式比get系列请求方式安全些，get系列的数据由于放在url地址上，有可能数据会被劫持

（2）xhr.readyState请求状态码
0 —- 创建ajax对象 unsent
1 —- 设置请求的地址，执行open后获取 opened
2 —- 响应头的内容返回 headers-receive
3 —- 响应主体的内容正在返回 loading
4 —- 响应成功，数据全部返回 done

模块化开发

高内聚是指一个软件模块是由相关性很强的代码组成，只负责一项任务，也就是常说的单一责任原则。
低耦合，粗浅的理解是：一个完整的系统，模块与模块之间，尽可能的使其独立存在。也就是说，让每个模块，尽可能的独立完成某个特定的子功能。模块与模块之间的接口，尽量的少而简单。如果某两个模块间的关系比较复杂的话，最好首先考虑进一步的模块划分。这样有利于修改和组合。
模块就是实现特定功能的一组方法。
只要把不同的函数（以及记录状态的变量）简单地放在一起，就算是一个模块。

JS盒模型

JS盒子模型

JS中提供了与盒子模型相关的属性
CSS盒子模型属性：width ， height ， border ， margin ， padding
13个属性中，只有与scrollLeft和scrollTop可以设置值，其他只能获取值

client系列(跟溢出的内容无关)
  • clientWidth = width+padding(左右)
  • clientHeight = height+padding(上下)
  • clientLeft 左边框
  • clientTop 上边框
    offset系列(跟溢出内容无关)
    • offsetWidth = width + padding(左右) + border(左右) = clientWidth + border(左右)
    • offsetHeight = height + padding(上下) + border(上下) = clientHeight + border(左右)
    • 与偏移量相关
  • offsetParent 参照物
  • offsetLeft 左偏移量
  • offsetTop 上偏移量
    scroll系列 (跟溢出内容有关)
  • scrollWidth ≈ width（获取真实的宽度）+ 左padding值 约等于真实宽度
  • scrollHeight ≈ height（获取真实的高度）+ 上padding值 约等于真实的高度
    为什么是约等于？
  • 各个浏览器的行高不一样
  • 相同浏览器是否设置overflow属性，值也不一样
  • 跟滚动条相关
    • scrollLeft 横向卷出去的内容
    • scrollTop 纵向卷出去的内容
